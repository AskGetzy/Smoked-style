Phase 1 — Install & Set Up

One-Time Setup
Phase 1

Install & Set Up

Do this once before you start building. Takes about 10 minutes. You only need to do it one time.

1
Download & Install Cursor
Go to cursor.com → click Download → run the installer → open Cursor when done.
2
Sign In to Cursor
When Cursor opens it will ask you to sign in. Create a free account or sign in with Google. The free plan is enough to start — you can upgrade later if needed.
3
Create Your Project Folder
Create a new empty folder on your Desktop called smoked-style. This is where all your site code will live.
4
Open the Folder in Cursor
In Cursor: click File → Open Folder → select the smoked-style folder you just created. Cursor will reload showing the empty folder.
5
Copy Your Environment File
Copy the file .env.local from C:\Users\Getzy\smoked claude\ into your new smoked-style folder. This file has your Supabase and Stripe keys — the app won't work without it.

Important: Never share this file or commit it to GitHub. It contains your live payment keys.
6
Open the Build Instructions
Open SmokedStyle_Cursor_Instructions.docx in Word or Google Docs. Keep it open alongside Cursor — you'll copy one prompt at a time from it as you build.
7
Open the Cursor Agent Chat
In Cursor, press Cmd+I (Mac) or Ctrl+I (Windows) to open the Agent panel on the right side. This is where you'll paste each prompt. Make sure the mode is set to Agent (not Chat or Edit).
Phase 2

Build Step by Step

19 prompts. One at a time. Check each step off as you complete it — your progress is saved in the browser.

⚠ Three Rules to Follow
  • One prompt at a time. Don't skip ahead — each step builds on the last.
  • Test before continuing. Open the browser after each step to make sure it works before moving on.
  • Be specific when something breaks. Don't tell Cursor to "fix everything" — describe exactly what's wrong and where.

Phase 3

Deploy to Vercel

Push your code to GitHub, then deploy to Vercel for a live public URL. Do this after completing all 19 build steps.

Before you start

Make sure all 19 build steps are complete and the app runs without errors locally (npm run dev in Cursor's terminal with no red errors).

1
Create a GitHub Account
If you don't have one, go to github.com → Sign Up → create a free account. GitHub stores your code in the cloud so Vercel can access it.
2
Add a .gitignore File
In Cursor's terminal (Ctrl+`), run: echo ".env.local" >> .gitignore echo "node_modules" >> .gitignore echo ".next" >> .gitignore This ensures your secret keys never get uploaded to GitHub.
3
Initialize Git and Make First Commit
In Cursor's terminal, run these commands one by one: git init git add . git commit -m "Initial commit"
4
Create a Private GitHub Repository
Go to github.com/new → Repository name: smoked-style → set to Private → click Create repository. Do not initialize with a README.
5
Push Your Code to GitHub
GitHub will show you two commands after creating the repo — copy and run them in Cursor's terminal. They look like: git remote add origin https://github.com/YOUR-USERNAME/smoked-style.git git push -u origin main Use the exact commands GitHub shows you (your username will be different).
6
Connect to Vercel
Go to vercel.com → click Sign In with GitHub → click Add New Project → click Import next to smoked-style.
7
Configure the Project
Vercel should auto-detect Next.js as the framework. Before clicking Deploy, expand the Environment Variables section and add all 7 keys:
Variable NameWhere to find it
NEXT_PUBLIC_SUPABASE_URLYour .env.local file
NEXT_PUBLIC_SUPABASE_ANON_KEYYour .env.local file
SUPABASE_SERVICE_ROLE_KEYYour .env.local file
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEYYour .env.local file
STRIPE_SECRET_KEYYour .env.local file
STRIPE_WEBHOOK_SECRETAdded in step 10 below
NEXT_PUBLIC_GOOGLE_MAPS_KEYPhase 2 feature — skip for now
8
Click Deploy
Click Deploy and wait about 2 minutes. When it finishes you'll see a live URL like smoked-style.vercel.app. Click it to open your live site. Save this URL — you'll need it for the next steps.
9
Update Supabase Redirect URL
Go to your Supabase dashboard → your project → Authentication → URL Configuration. Add your Vercel URL to Redirect URLs: https://your-site.vercel.app/** This allows Google OAuth login to work on your live site.
10
Add Stripe Webhook
Go to Stripe Dashboard → Webhooks → Add endpoint. Set the endpoint URL to: https://your-site.vercel.app/api/webhooks/stripe Select events: payment_intent.succeeded, payment_intent.payment_failed, charge.captured. Copy the Signing secret that appears. Add it to Vercel as STRIPE_WEBHOOK_SECRET and redeploy.
11
Test End-to-End
On your live site, test the full flow:
  • Browse the catalog and add an item to cart
  • Go through checkout with a real card
  • Check that the order appears in /admin
  • Approve the order — verify the card gets charged in Stripe dashboard
  • Log in to admin panel and confirm auth works
Phase 4

After Launch

Your site is live. Here's how to keep it running and updated.

🔄
How to Update the Site
Make changes in Cursor, then run in the terminal:

git add .
git commit -m "describe what changed"
git push

Vercel automatically detects the push and redeploys in ~1 minute. No manual action needed.
👤
How to Add Admin Users
Go to your Supabase Dashboard → your project → Authentication → Users → click Invite User.

Then in the database, add their email to the admin_users table so the admin panel grants them access.
📦
Where to Monitor Orders
Go to your-site.vercel.app/admin — this is your live admin panel. You'll see incoming orders in real-time, can approve or reject them, and manage inventory from there.
💳
Monitor Payments in Stripe
Go to Stripe Dashboard → Payments.

Authorized payments show as Uncaptured. When you approve an order in the admin panel, the capture happens automatically. Failed webhooks appear under Webhooks → Logs.
🛑
If Something Breaks
Check Vercel Dashboard → your project → Deployments → Functions for server errors.

For database issues, check Supabase → Logs → API. Paste the error into Cursor Agent and describe what you were doing when it happened.
🗺️
Google Maps (Phase 2)
The NEXT_PUBLIC_GOOGLE_MAPS_KEY in your .env.local is a placeholder. When you're ready to add driver routing and route tracking, get a key from Google Cloud Console and add it to Vercel's environment variables.
⚠ Live Keys Reminder

Your Stripe keys are live production keys — any charges made on your site are real charges. Always test with a real card you own before going public. Never commit .env.local to GitHub.
